# Architecture Decision Records

This folder stores Architecture Decision Records (ADRs) for the SaaSCoop Engineering Platform.

ADRs document important decisions, their context, alternatives, consequences, risks, and mitigations.

## Naming Convention

Use the following pattern:

```text
ADR-000-short-title.md
ADR-001-adopt-saascoop-organization.md
ADR-002-repository-structure.md
```

## Status Values

- Proposed
- Accepted
- Superseded
- Rejected

## When to Create an ADR

Create an ADR when a decision affects:

- architecture
- governance
- repository structure
- licensing
- security
- data strategy
- AI engineering standards
- tooling
- deployment
- community contribution model
